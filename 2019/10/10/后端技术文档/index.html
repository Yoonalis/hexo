常用工具类介绍

CodeGenerator

根据数据库表名生成，生成dao，serveice，controller 层代码，使用时只需修改 MODULE_NAME（模块名）、AUTHOR（作者名）、TABLES（数据库表名），修改完后运行即可。

Constant

常量，统一存放后端代码中出现的常量

GlobalExceptionAdvice

全局异常拦截器

ViewException

RuntimeException，被GlobalExceptionAdvice拦截，处理后将错误信息在前端展示

Page

分页查询 current 第几页，size 当前页的大小

GridRequest

封装前端查询请求，Page（分页请求）、filterParams（过滤参数 k:v）、searchParams（查询参数 k:v），首先根据filterParams 过滤，多个过滤条件同时满足（and），然后根据searchmentParams 进行查询，多个查询条件是或连接（or），最后返回分页后的数据

GridResponse

封装数据库返回结果，total 当前查询总记录条数，records 具体的记录

Result

封装后端返回给前端的数据，code （状态码），msg （信息），data （数据负载）

ResultEnum

常用Result枚举

ResultUtil

快速返回Result工具

GridService

getGridResponse() 快速单表查询，根据gridRequest 的分页，过滤，查询条件，返回对应表的数据

getGridResponseWithInParam() 满足以上条件，再加上in 过滤 （通常用于多表）

getGridResponseWithNotInParam 满足以上条件，再加上not in 过滤 （通常用于多表）

UploadEntity

封装了上传的文件，创建时会进行文件合法性检查

suffixes（合法的文件后缀）

files （上传多个文件时使用）

file （上传单个文件时使用）

数据库设计

命名

数据库名：用下划线连接，例exam_question、training_question。关联表也用下划线连接 例：user_exam

字段名：用下划线连接，例user_type

多表删除

被关联表中加一个status状态、status == 0 代表记录被删除，status == 1 表示状态正常，逻辑删除，这样可以增加处理速度。真正的物理删除可以使用触发器，定时器，或者管理员定时清理

bucket命名规则

The following rules apply for naming buckets in Amazon S3:

  • Bucket names must be between 3 and 63 characters long.
  • Bucket names can consist only of lowercase letters, numbers, dots (.), and hyphens (-).
  • Bucket names must begin and end with a letter or number.
  • Bucket names must not be formatted as an IP address (for example, 192.168.5.4).
  • Bucket names must not start with the prefix xn--.
  • Bucket names must not end with the suffix -s3alias. This suffix is reserved for access point alias names. For more information, see Using a bucket-style alias for your access point.
  • Bucket names must be unique within a partition. A partition is a grouping of Regions. AWS currently has three partitions: aws (Standard Regions), aws-cn (China Regions), and aws-us-gov (AWS GovCloud [US] Regions).
  • Buckets used with Amazon S3 Transfer Acceleration can’t have dots (.) in their names. For more information about Transfer Acceleration, see Configuring fast, secure file transfers using Amazon S3 Transfer Acceleration.
