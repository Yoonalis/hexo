JVM

1: jvm 组成部分，运行流程
答：jvm 由类加载子系统，运行时数据区，执行引擎，本地方法库 组成。

2：jvm 的运行时数据区
jvm 的运行时数据区由程序计数器，虚拟机栈，本地方法栈，方法区，堆几部分组成
程序计数器，记录了下一条虚拟机指令的位置
虚拟机栈是由一个一个栈帧组成，每当调用一个方法就会把一个栈帧入栈，栈帧中包含了局部变量表，操作数栈，方法出口，动态链接等信息
本地方法栈与虚拟机栈类似，不过本地方法栈服务于Native 本地方法
方法区 用于存放类相关信息，常量，静态变量等
堆 用于存放对象实例
程序计数器，虚拟机栈，本地方法栈 是线程私有的
方法区，堆是线程共享的

3：如何判断对象能否被垃圾回收？
引用计数法：每个对象持有一个引用计数器，当其他对象引用该对象时引用计数 +1，当引用计数为0时，回收该对象。会出现循环引用的问题，
假如此时A引用B，B引用A，没有其他引用引用A，B， A，B 的引用计数为1，不会被清楚
可达性分析法：GCRoot法，从GcRoot 出发，不在GcRoot 引用链上的对象被视作可回收对象
GCRoot由这几部分组成：
1：系统类对象 2：本地对象 3：加锁对象 4：活动线程对象

4：4种引用
强，软，弱，虚
强：一般用的所有引用都是强引用，有强引用引用对象时，对象不会被回收
软：软引用引用对象时，发生垃圾回收后空间仍不足时，对象才会回收
弱：弱引用引用对象时，只要发生垃圾回收，对象都会被回收
虚：虚引用引用对象时，对象在垃圾回收后，会检查对象是否有虚引用，如果有，会把虚引用加入引用队列 例如ByteBuffer 对象置为null，后会将一个Clean
对象置入引用队列，然后由一个ReferenceHandler 从引用队列取出虚引用Clean ，执行Clean 当clear 方法 ，间接调用Unsafe 的freeMemory 方法
释放直接内存

5：垃圾回收算法
标记清除算法，复制算法，标记整理算法，分代回收算法
标记清除算法：标记要被回收的对象，清除，会产生内存碎片
复制算法：标记存活对象，将对象复制到另一块区域中区
标记整理算法：将存活的对象向一端移动，然后清除掉端边界以外的内容

6：分代回收算法
将堆分成新生代，和老年代，其中新生代由Eden Survival From Survival To 三部分组成，默认比例8：1：1
对象首先分配到Eden区，当Eden区内存不足时，触发Mirror Gc，将Eden 区，From 区中存活对象复制到To 区，存活年龄+1
然后清除Eden ，From区，交换From To 区指针， 当存活年龄到达一定阈值（最大15），对象就会晋升到老年代
当老年代空间不足时，会触发Full GC 在此之前会触发Major Gc，Full Gc 会清理新生代，老年代的垃圾

7：说一下Jvm 有哪些垃圾回收器
大致分为串行，吞吐量优先，响应时间优先 三种
串行 Serial，Serial Old
分别工作在新生代和老年代，单线程垃圾回收，适用于单核Cpu
吞吐量优先
ParNew,Parallel Scavenge,Parallel Old
其中ParNew，Parallel Old 分别时Serial，Serial Old 的并行版本
Parallel Scavenge 目的是为了达到可控吞吐量
响应时间优先
Cms
Cms 工作在老年带，采用标记清除算法，分为初始标记，并发标记，重新标记，并发清理几个过程
初始标记阶段会标记那些存活的GcRoot，会造成STW
并发标记阶段从GcRoot 出发标记剩下存活对象，与用户线程并发运行，不会造成STW
重新标记阶段 消除并发标记阶段新产生垃圾等影响，会造成STW
并发清理阶段与用户线程并发运行，不会造成STW

G1
即重视吞吐量，又重视响应时间
把堆分成一个个Region，采用复制算法+标记整理算法，分为三个阶段，Young Collection，Young Collection + CM ， Mixed Collection
Young Collection 阶段 当Eden Region 内存空间不足时，把Eden Region ，Survival Region
存活对象0复
制14到 Survival Region，存活年龄 +1
Young Collection + CM 阶段 当老年代占用达到这个堆当一定比例时，在进行Young Collection时进行 并发标记
Mixed Collection 当并发标记完成后进行混合收集，新生带执行YoungCollection 老年代挑选一些回收价值高的Region 进行回收

8：java 类加载机制
加载 链接（验证，准备，解析） 初始化
加载：将字节码文件加载到方法区
验证：验证类是否符合Jvm规范
准备：为静态变量分配内存空间并赋默认值
解析：符号引用解析为直接引用
初始化：初始化静态变量

9：类加载器
启动类加载器：加载 %JAVA_HOME%/lib 下的类
扩展类加载器：加载 %JAVA_HOME%/jre/lib/ext 下的类
系统类加载器：加载 %ClassPath% 下的类
用户类加载器：加载用户自定义的类

10：什么是双亲委派机制，为什么要双亲委派机制
双亲委派机制：类加载器加载类时，会先委派父类加载器加载类，只有当父类加载无法完成加载时，才轮到子类加载器加载
好处：
1：保护java 核心类api，更加安全 2：避免重复加载

11：jvm 调优
调优工具：
jStack：检测线程
jMap：检测堆
jConsole：对堆，线程，对象进行监控
jvisualvm

调优参数：
-Xms：初始堆大小，JVM 启动的时候，给定堆空间大小。

-Xmx：最大堆大小，JVM 运行过程中，如果初始堆空间不足的时候，最大可以扩展到多少。

-Xmn：设置堆中年轻代大小。整个堆大小=年轻代大小+年老代大小+持久代大小。

-XX:NewSize=n 设置年轻代初始化大小大小

-XX:MaxNewSize=n 设置年轻代最大值

-XX:NewRatio=n 设置年轻代和年老代的比值。如: -XX:NewRatio=3，表示年轻代与年老代比值为 1：3，年轻代占整个年轻代+年老代和的 1/4

-XX:SurvivorRatio=n 年轻代中 Eden 区与两个 Survivor 区的比值。注意 Survivor 区有两个。8表示两个Survivor :eden=2:8 ,即一个Survivor占年轻代的1/10，默认就为8

-Xss：设置每个线程的堆栈大小。JDK5后每个线程 Java 栈大小为 1M，以前每个线程堆栈大小为 256K。

-XX:ThreadStackSize=n 线程堆栈大小

-XX:PermSize=n 设置持久代初始值

-XX:MaxPermSize=n 设置持久代大小

-XX:MaxTenuringThreshold=n 设置年轻带垃圾对象最大年龄。如果设置为 0 的话，则年轻代对象不经
